■ 폴더 구성
+ 스톡캐쳐_VOL22.html ........ 웹앱 본체 (Netlify에 이 파일만 배포해도 됨)
+ pipeline/collect_krx.py ... 전 종목 수집 + 지표 계산 (VOL15 파이프라인: 46종 지표 + 수급 + 종목별 3년 차트, 시가0가드·수급120일)
+ .github/workflows/daily.yml 매일 자동 실행 설정 (평일 19:00 KST)
+ data/ ..................... screener.json이 생성되는 위치
+ data/charts/ .............. 종목별 차트 데이터(120거래일 OHLCV)가 생성되는 위치

■ 설치 순서 (약 15분, 전부 무료)

1️⃣ GitHub에 새 저장소 생성 (Public 권장 — Actions 무료 무제한)

2️⃣ 이 폴더 내용물을 그대로 저장소에 업로드
   (.github 폴더가 숨김 처리되지 않게 주의)

3️⃣ 저장소 Settings → Actions → General →
   Workflow permissions를 "Read and write permissions"로 변경 후 저장

4️⃣ Actions 탭 → "스톡캐쳐 일일 데이터 수집" → Run workflow 버튼으로
   첫 수집을 수동 실행 (약 40~50분 소요 — 3년치 시세 + 수급 수집 포함.
   워크플로우 제한시간은 120분으로 넉넉히 설정되어 있어요)

5️⃣ 완료되면 data/screener.json 생성 확인 후, 파일 화면에서
   Raw 버튼 주소 복사:
   https://raw.githubusercontent.com/계정명/저장소명/main/data/screener.json

6️⃣ 스톡캐쳐 화면 우측 상단 "🧪 데모 시세" 배지 클릭 → 주소 붙여넣기
   → 새로고침되며 "📡 실데이터 연결됨"으로 전환

이후에는 평일 저녁마다 자동 갱신돼요. 수동 갱신이 필요하면
Actions 탭에서 Run workflow를 누르면 돼요.

■ VOL17 핫픽스 안내 (수정 2건)
+ 종목 클릭 시 화면이 빈 화면(검은 배경)이 되는 오류 — 지표 계산 함수
  (calcSeries)가 리팩토링 과정에서 누락된 것으로, 원본을 복원해 해결했어요.

  (setCustom({}) → setCustom([]))를 바로잡아 해결했어요.
+ 검증 범위: 탐색/스코어/관심종목 탭, 시장 필터, 프리셋, 채팅 조건검색,
  초기화, 차트 열기, 일/주/월봉 전환, 1/2/3년 전환, 오버레이 8종,
  보조차트 12종, 이평선 기간 추가/삭제, 십자선, 수평선/추세선 그리기,
  관심종목 담기/해제 — 전 항목 실제 브라우저 구동으로 통과 확인.

■ VOL21 신규 기능 — 💰 수급 대시보드 + 보조지표 접힘
+ 수급 대시보드 탭 — 상단 탭에 "💰 수급"이 추가됐어요. 투자자별
  (외국인 / 기관 / 연기금[국민연금 등])로 기간 단위(1일·1주·1개월·3개월)를
  바꿔가며 순매수 상위 20종목·순매도 상위 20종목을 볼 수 있어요.
+ 종목마다 업종(철강금속, 전기전자, 서비스업 등 KRX 업종분류)이 붙고,
  비중 2가지를 함께 보여줘요: "시총比 %"(그 종목 시가총액 대비 순매수 규모)와
  "%"(그 투자자의 기간 순매수 총액 중 이 종목 비중). 스크리너 대상 종목은
  클릭하면 바로 차트가 열려요.
+ 업종(분야)별 자금 흐름 — 투자자·기간 기준으로 어느 업종에 돈이 들어오고
  (▲유입) 어느 업종에서 빠지는지(▼유출)를 막대로 집계해요.
+ 국민연금 표기 — KRX는 국민연금 단독 수치를 공개하지 않아서, 국민연금이
  대부분을 차지하는 '연기금' 집계를 쓰고 화면에 그렇게 정직하게 표기해요.
+ 데이터 — pipeline/flow_krx.py 가 신설되어 일일 수집 워크플로우에 자동
  포함됐어요(data/flows.json 생성, 추가 소요 약 1~2분). 별도 설정은 없고
  최신 daily.yml + flow_krx.py 반영 후 수집 1회면 대시보드가 채워져요.
+ 보조지표 기본 접힘 — 차트 모달이 길다는 의견을 반영해 보조차트는 이제
  거래량만 기본 표시되고 나머지는 접혀 있어요. "▼ 모두 펼치기 / ▲ 모두 접기"
  버튼으로 한 번에 조절할 수 있고, 선택 상태는 저장돼요.

■ VOL20 신규 기능 — 상장 이후 '전체' 데이터 + 패닝/핀치
+ 과거 데이터 백필(자동 확장) — 새 워크플로우 "스톡캐쳐 과거 데이터 백필"이
  매일 21:30(KST) 자동 실행되며, 실행할 때마다 미완료 종목의 과거 데이터를
  상장 시점까지 내려받아요. 시간 예산(기본 300분) 안에서 처리하고, 다음 실행이
  이어서 진행돼 결국 전 종목이 완료돼요. Actions에서 Run workflow로 수동 실행도
  가능하고, codes 입력에 종목코드를 넣으면 그 종목을 우선 처리해요.
+ 저장 구조 — 완성된 과거분은 data/charts_hist/{코드}.json 에 '한 번만' 기록되고
  이후 다시 건드리지 않아 저장소가 매일 불어나지 않아요. 매일 갱신은 기존
  롤링 3년 파일이 담당하고, 앱이 두 파일을 날짜 기준으로 이어 붙여요.
+ 앱 표시 — 차트 상단에 종목별 상태가 떠요:
    📥 상장 이후 전체 확보 완료 · 총 N봉(약 X년)  ← 그 종목 다운로드 완료 표시
    ⏳ 과거 확장 중 / 과거 확장 대기               ← 백필이 돌 때마다 자동 확장
  기간 버튼에 "전체"가 추가되어 확보된 전 구간을 한 번에 볼 수 있어요.
+ 드래그 패닝 — 차트를 좌우로 끌면 과거/최신으로 이동해요 (1/2/3년 뷰에서도
  왼쪽으로 끌면 그 이전 데이터가 보여요). 드래그 직후 클릭은 십자선으로
  오인되지 않아요.
+ 모바일 핀치 줌 — 두 손가락을 벌리면 확대, 오므리면 축소돼요. 한 손가락
  좌우 드래그는 패닝, 세로 스크롤은 그대로 페이지 스크롤로 동작해요.
+ 주의 — 일일 수집기(collect_krx.py)에 날짜 배열이 추가되어, 백필 과거분과의
  연결은 새 수집기로 일일 수집이 1회 이상 실행된 뒤부터 활성화돼요.

■ VOL19 신규 기능 3가지
+ 마우스 휠 줌 — 차트 위에서 휠을 올리면 커서 위치 중심으로 확대,
  내리면 축소돼요(최소 15봉 ~ 전체 데이터). 확대 중에는 우측 상단에
  "🔍 확대 N봉 / 전체 N봉"과 ⟲ 초기화 버튼이 표시되고, 기간·주기를
  바꾸면 자동으로 초기화돼요. 차트 위에서는 페이지가 스크롤되지 않아요.
+ 일·주·월 평균 거래 — 거래량 패널 아래에 일평균(최근 20거래일),
  주평균(최근 12주), 월평균(최근 6개월) 거래량과 거래대금을 함께 보여줘요.
  주·월은 환산이 아니라 완결된 5거래일/20거래일 합계의 실제 평균이에요.
+ 장기 이평선 안내 — 3년 뷰에서 120·240일선이 앞부분에 보이지 않는 건
  오류가 아니라 계산에 그만큼의 봉이 필요해서예요. 이제 범례에
  "◔ 120일·240일선은 데이터 시작 후 그만큼 지나야 그려져요"라고
  이유를 표시해 오해가 없도록 했어요.

■ VOL18 신규 기능 4가지
+ 왜 이 종목이 걸렸나요? — 검색 조건(지표·나만의 조건·채팅 검색)을 적용하고
  종목을 열면, 충족한 조건마다 구체적 수치 근거를 설명하고 발생 지점을
  차트에 번호 표식(●)과 점선으로 표시해요. 미충족 조건도 함께 알려줘요.
+ 보조지표 읽기 — 켜져 있는 보조차트마다 지금 값이 무엇을 의미하는지
  한 줄 해설을 차트 아래에 보여줘요. 보조차트를 켜고 끄면 해설도 바뀌어요.
+ 이평선 개선 — 120일·240일 등 추가한 이평선도 항상 서로 다른 색으로
  배정되고, 각 선의 오른쪽 끝에 몇 선인지 라벨이 붙어요 (주봉이면 N주선).
+ 진짜 박스권 감지 — 단순 고점·저점이 아니라 ①폭 25% 이내 ②종가 92% 이상
  수용 ③추세 드리프트 ±7% 이내 ④저항·지지 각 2회 이상 반복 터치를 모두
  검증해 감지된 구간만 박스로 표시해요. 터치 지점(▼▲)·폭·기울기·터치 횟수를
  차트와 범례에 근거로 보여주고, 조건을 못 채우면 "박스권 미감지 — 참고용
  고저 범위"라고 정직하게 표시해요. 최근 상단 돌파/하단 이탈도 알려줘요.
